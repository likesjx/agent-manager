# Contributing

## Development setup

1. Use Node 20+.
2. Clone repo and run:
   - `npm run validate`

## Pull requests

- Keep changes scoped and include rationale in the PR body.
- Update docs when behavior or contracts change.
- Add or update adapter examples when touching intake/provider code.

## Commit guidance

- Prefer clear, imperative messages:
  - `Add iTrack cursor-based intake sync`
  - `Document ADO environment variables`
